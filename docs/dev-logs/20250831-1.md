### 技术决策

#### 1. 构建工具选择
- **决策**: 选择 tsup 而非 tsc
- **原因**: tsup基于esbuild，构建速度更快，配置更简单
- **考虑**: 虽然学习成本略高，但长期收益明显

#### 2. 架构模式
- **决策**: 采用事件驱动 + Promise 异步架构
- **原因**: 
  - 支持高并发任务处理
  - 便于扩展和维护
  - 符合Node.js生态习惯
- **实现**: TaskManager负责调度，各模块通过Promise通信

#### 3. 模型集成策略
- **决策**: 优先支持 DeepSeek，预留其他模型扩展接口
- **原因**: 
  - 项目目标明确针对DeepSeek
  - 通用接口设计便于后续扩展
  - ModelManager提供统一抽象层

#### 4. 安全机制设计
- **决策**: 实现6层权限验证 + 沙盒隔离
- **原因**: 
  - AI工具需要严格的安全控制
  - 分层验证提供细粒度控制
  - 沙盒隔离防止恶意代码执行

#### 5. CLI设计原则
- **决策**: 参考 Claude Code 的命令模式和交互体验
- **原因**: 
  - 用户熟悉度高
  - 功能完整性好
  - 扩展性强

## ⚠️ 遇到问题

### 问题1: TypeScript类型错误
- **问题描述**: `import type` 导入的类型不能作为值使用
- **解决方案**: 区分类型导入和值导入，修改import语句
- **相关文件**: 多个核心文件的import语句
- **教训**: 严格区分类型和值的导入很重要

### 问题2: tsup构建配置
- **问题描述**: CLI shebang重复导致构建失败
- **解决方案**: 
  - 移除源文件中的shebang
  - 配置tsup只在CLI文件添加shebang
- **相关文件**: `tsup.config.ts`, `src/cli.ts`

### 问题3: inquirer类型兼容性
- **问题描述**: inquirer v12的类型定义变化导致编译错误
- **解决方案**: 调整prompt调用方式，移除不支持的prefix配置
- **相关文件**: `src/cli/commands.ts`

### 问题4: 测试Mock配置
- **问题描述**: axios mock在测试中表现异常
